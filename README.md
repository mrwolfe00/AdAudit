# ADAudit Work Hours Viewer

A small in-browser dashboard for the **User Work Hours** report exported from
ManageEngine ADAudit Plus. Pulls each user's per-day totals, groups them
Mon&ndash;Sun with weekly totals, surfaces violations (short Mon&ndash;Thu days,
failed Friday half-days), and shows a Top 10 leaderboard. Optionally maps users
to their AD department.

Everything runs **client-side** &mdash; the xlsx is parsed in the browser via
SheetJS, nothing is uploaded anywhere.

## Quick start

1. **Open the app** &mdash; double-click `ADAudit Work Hours` on your Desktop, or
   run `npx -y serve -l 4333 C:\git\adaudit-tool` and visit
   `http://localhost:4333`.
2. **Drop the .xlsx** &mdash; the ADAudit "User Work Hours" export. Any filename
   ending in `.xlsx` works.
3. **(Optional) Map departments** &mdash; run `Update-Departments.ps1` to pull
   `samAccountName` + `department` from your domain controller. Drop the resulting
   `departments.csv` onto the app, or open via `localhost` for auto-load.

## Files

| File | What it does |
|---|---|
| `index.html` | UI shell |
| `app.js` | Parsing, rendering, departments, attention/earned tabs |
| `styles.css` | All styles |
| `vendor/xlsx.full.min.js` | SheetJS, bundled so the app works offline |
| `Update-Departments.ps1` | Queries AD, writes `departments.csv` |
| `departments.csv` | Generated by the PS script. Not in git &mdash; contains org data |

## The rules it enforces

| Rule | What happens |
|---|---|
| Day total &lt; 9h on Mon, Tue, Wed, or Thu | Cell turns **amber** |
| 9+ hours each day Mon&ndash;Thu | Gold &#9733; next to the username (earned half-day Friday) |
| Friday &lt; 4 hours (anyone) | Cell turns **red** |

## Tabs

- **Overview** &mdash; Top 10 by weekly total + the full users-by-days grid with
  Department column, weekly Active/Idle/Total, sort/filter, CSV export.
- **Needs Attention** &mdash; two grouped lists: users with at least one short
  Mon&ndash;Thu day, and starred users who failed the 4-hour Friday minimum.
  Each row shows department and severity.
- **Earned Half-Day** &mdash; all starred users with their Mon&ndash;Thu hours,
  Friday actual, and a &#10003; OK / &#10005; Short status.

## Where data lives

- **Work hours xlsx** &mdash; re-uploaded each session (never persisted)
- **Departments CSV** &mdash; stored in browser localStorage so it survives
  closing the browser. Re-load via drag/drop or via `departments.csv` next to
  `index.html` when served from `localhost`
- **Settings / state** &mdash; none. Everything else is recomputed from the
  uploaded xlsx
